# принципы и подход к работе

* не придумываем решения на ходу, сперва изучаем проблему и контекст задачи
* крайне внимательно осознаем задачу, если задача не ясна полностью - уточняем

# кодстайл и форматтеры

* простой, человекочитаемый код
* не используем deprecated методы
* для svelte предпочитаем реактивность (svelte style). Если реактивность тянет за собой сложность кода - используем функции.

# архитектурные принципы, слои
* UI и логику разносим в разные файлы
* типы описываем в frontend/src/lib/types.ts
* * можно расширять типы, сохраняя смысловую составляющую
* * не нужно плодить типы данных на каждый случай, балансируем с простотой кода
* все тексты на UI не забываем локализовать (i18n). В локализации группы плейсхолдеров формируем по принципу "компонент приложения - группа"


# правила ошибок и логирования
* логируем в консоль минимально или для отладно. Отладочные логи не забываем удалять.
